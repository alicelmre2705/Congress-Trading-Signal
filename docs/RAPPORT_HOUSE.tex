\section{Résumé exécutif}
La phase House est terminée. On extrait les déclarations boursières (PTR, STOCK Act) des membres de
la Chambre des représentants, 2020--2026. Deux pistes : \textbf{électronique} (PDF lisibles, parsing
déterministe) et \textbf{OCR} (PDF scannés, Claude Vision). Le résultat est une table unique de
\textbf{82\,051 transactions}.

Quiver Quantitative sert de vérification externe, jamais de source. La concordance FINAL contre
Quiver est de \textbf{85,9\,\%} au niveau transaction. Le numérique seul plafonne à 35--74\,\% : c'est
l'OCR qui ferme l'écart. Après explication des artefacts (tickers Quiver, amendements, dates $\pm$1\,j),
seules \textbf{9 transactions} sur 82\,051 sont réellement absentes. L'identité (BioGuideID) est à
\textbf{99,99\,\%}. Le code est factorisé dans un cœur partagé \texttt{congress\_core} réutilisable par
le Sénat, avec un filet de non-régression de 108 fichiers.

\section{Objectif et périmètre}
\textbf{But.} Construire une stratégie de \emph{copy-trading} répliquant les transactions déclarées
par le Congrès, puis la backtester. Le présent rapport couvre la \textbf{couche données House}.

\textbf{Sources.} Primaire : House Clerk (\texttt{disclosures-clerk.house.gov}, index XML + PDF).
Vérification : Quiver (licence R\&D, jamais réinjecté). Référentiel identité :
\texttt{unitedstates/congress-legislators}.

\textbf{Périmètre.} House, 2020--2026, électronique \emph{et} scanné. \texttt{disclosure\_date} =
\emph{FilingDate} de l'index (anti look-ahead). Hors périmètre : Sénat (à venir), stratégie et backtest.

\textbf{Plafond de complétude annoncé.} « Ère électronique niveau Member, complète et vérifiée ; OCR
des scans tapés (clusters A+B) complet ; manuscrit (C) récupéré de façon ciblée ; chaque exclusion
comptée et expliquée. »

\section{Pipeline étape par étape}
\subsection{Piste électronique (\texttt{house/digital.py})}
\begin{enumerate}[nosep,leftmargin=1.5em]
\item \textbf{Référentiel} : \texttt{build\_reference} (délégué au cœur) $\rightarrow$ 12\,767 législateurs.
\item \textbf{Index} : \texttt{load\_ptr\_index} filtre \texttt{FilingType=P} + fenêtre $\rightarrow$ \texttt{03\_ptr\_index}.
\item \textbf{Manifest} : \texttt{build\_manifest} route lisible/non\_lisible/absent $\rightarrow$ \texttt{04\_download\_manifest}.
\item \textbf{Parsing} : \texttt{parse\_ptr} (regex marqueurs) ; échecs $\rightarrow$ \texttt{05\_parse\_failures}. Rendement 99--100\,\%.
\item \textbf{Identité} : \texttt{join\_identity} $\rightarrow$ bioguide, parti, commissions.
\item \textbf{Finalize} : \texttt{natural\_key\_hash} + \texttt{occurrence\_index} (dédup per-lot) $\rightarrow$ \textbf{\texttt{06\_transactions}} (digital). \textbf{32\,676 txns}.
\item \textbf{Cross-check} baseline $\rightarrow$ \texttt{08}. \textbf{Quiver} : \texttt{validate\_quiver\_house} $\rightarrow$ \texttt{07} + \texttt{07b}.
\end{enumerate}

\subsection{Piste OCR (\texttt{house/ocr.py})}
\begin{enumerate}[nosep,leftmargin=1.5em]
\item \textbf{Sélection} : scannés (\texttt{non\_lisible}) $\cap$ census. Cluster C exclu par défaut (sauf 3 filers ciblés).
\item \textbf{Deskew} : pré-passe Vision « montage 4-rotations », page par page, redressement PIL.
\item \textbf{Vision} : Claude \texttt{sonnet-4-6}, \texttt{tool\_use} forcé, cache versionné (\texttt{PROMPT\_SHA}, reprise au batch).
\item \textbf{Normalize} : montant A--K, owner, \texttt{date\_confidence} (fenêtre légale 75\,j), filtre ligne-exemple.
\item \textbf{Ticker} : explicit $\rightarrow$ dict électronique $\rightarrow$ passe LLM (cache global). $\rightarrow$ \textbf{\texttt{06b}}. \textbf{49\,375 txns}.
\item \textbf{Fusion} : digital + OCR, dédup inter-sources (collision hash) $\rightarrow$ \textbf{\texttt{06\_FINAL}}. \textbf{82\,051 txns}.
\item \textbf{Quiver OCR} : \texttt{validate\_ocr\_quiver} $\rightarrow$ \texttt{06d}.
\end{enumerate}

\section{Validation : concorde-t-on bien ?}
\textbf{Méthode Quiver honnête.} Deux axes : (1) comptes par déclarant \emph{per-lot} (Quiver brut,
déposants papier flaggés) ; (2) recouvrement au niveau transaction (clé invariante à la dédup). Quiver
n'entre jamais dans la table.

\textbf{Verdict.} Oui, le FINAL concorde bien. Couverture 85,9\,\% (81--91\,\%/an), accord sens
$\sim$93\,\%, montant $\sim$93\,\%, identité $\sim$100\,\%, et \textbf{9 transactions réellement
absentes sur 82\,051}. Le numérique seul est faible (35--74\,\%) ; l'OCR ferme l'écart. Le papier
(Khanna, Harshbarger\dots) n'a aucune source externe : il est validé en interne (PDF source,
\texttt{date\_confidence}, stabilité run-à-run, ticker\,\%).

\section{Décisions et justifications}
\begin{enumerate}[nosep,leftmargin=1.5em]
\item \textbf{Quiver = vérification seule.} Garantit l'indépendance ; nos chiffres ne sont pas tirés vers Quiver.
\item \textbf{Dédup per-lot (\texttt{occurrence\_index}).} Préserve les lots identiques légitimes d'un même PTR ; retire seulement les vrais doublons cross-dépôt.
\item \textbf{\texttt{natural\_key\_hash} = 7 champs sans ticker.} Robuste à l'enrichissement (le ticker arrive après).
\item \textbf{Cluster C non exécuté par défaut.} Dates manuscrites peu fiables ; découpage par qualité, pas par année ; récupération ciblée des 3 filers à forte perte Quiver.
\item \textbf{Deskew.} 525/547 scans couchés sans signal métadonnée ; montage 4-rotations (reconnaissance, pas calcul d'angle), page par page. Concordance 59,7\,\%$\rightarrow$69,0\,\%.
\item \textbf{\texttt{date\_confidence}.} Flag indépendant de Quiver ; \texttt{implausible} = hors fenêtre légale (misread d'année). Attrape les dates aberrantes.
\item \textbf{Passe LLM nom$\rightarrow$ticker.} Le dict électronique plafonne à $\sim$46\,\% ; LLM monte à $\sim$90\,\%, cache versionné (re-run gratuit).
\item \textbf{Zéro changement de résultat.} La refonte ne modifie aucun octet de sortie ; prouvé fonction par fonction.
\item \textbf{Arbo plate \texttt{data/house/}.} Structure présentable ; déplacement vérifié (\texttt{check\_golden} = 0 écart).
\item \textbf{Maps montant/owner/opération par chambre.} House et Sénat divergent (midpoints, labels) ; centralisé dans \texttt{amounts.py}.
\item \textbf{\texttt{PROMPT\_SHA} préservé.} Déplacer le moteur OCR ne devait pas invalider 546 caches (re-OCR payant) ; prouvé.
\item \textbf{Clés Quiver non fusionnées entre chambres.} House garde sa clé brute (reproduit \texttt{07}) ; Sénat la sienne.
\item \textbf{Tolérance date $\pm$3\,j (échantillon).} Quiver bruite la date de $\pm$1--2\,j ; l'égalité stricte comptait ces écarts comme erreurs.
\item \textbf{Override manuel (Van Taylor).} « Nicholas V. Taylor » = Van Taylor, résolu explicitement.
\item \textbf{Suppression des globals identité House.} \texttt{build\_reference}/\texttt{match\_bioguide} délèguent au cœur ; plus aucune duplication.
\end{enumerate}

\section{Limites et métriques manquantes}
\begin{enumerate}[nosep,leftmargin=1.5em]
\item \textbf{Validation papier externe impossible.} Quiver, Kadoa et House Stock Watcher sautent tous les scans : 0 ligne pour Khanna/Harshbarger/McCaul. Pas de taux d'exactitude externe sur le papier (contrôles internes seulement).
\item \textbf{Comités non point-in-time.} \texttt{committee\_membership} est constant sur 7 ans (0/256 varient) : snapshot statique. Une analyse de conflit d'intérêt par comité à la date de transaction n'est pas fondée.
\item \textbf{Pas de tolérance de date dans l'appariement House.} Clé date brute : une ambiguïté OCR d'1 jour casse le match. La couverture 85,9\,\% est donc un \emph{plancher} (sous-estime sur dates OCR bruitées).
\item \textbf{\texttt{only\_quiver} non décomposé par cause.} 248 brut : on sait que 9 sont réellement absents, mais le reste (amendements / artefacts ticker / dates) n'est pas ventilé automatiquement.
\item \textbf{Divers.} \texttt{asset\_type} rempli à 91,1\,\% ; \texttt{date\_confidence} est une heuristique binaire (plausible/implausible), pas une mesure d'exactitude vs vérité terrain.
\end{enumerate}

\subsection{Glossaire}
\begin{description}[nosep,leftmargin=0em,style=nextline]
\item[PTR] Periodic Transaction Report : déclaration de transaction (STOCK Act, $\leq$45\,j).
\item[BioGuideID] Identifiant unique d'un législateur (référentiel \texttt{congress-legislators}).
\item[natural\_key\_hash] SHA-256 de 7 champs (sans ticker) : clé de transaction stable.
\item[per-lot] Dédup qui préserve les lots identiques d'un même PTR via \texttt{occurrence\_index}.
\item[deskew] Redressement des scans couchés avant l'OCR (montage 4-rotations).
\item[date\_confidence] Flag plausible/implausible d'une date OCR (fenêtre légale 75\,j).
\item[cluster A/B/C] Typologie des scans : tapé droit / tapé tourné / manuscrit.
\item[only\_ours / only\_quiver] Transactions présentes chez nous seulement / chez Quiver seulement.
\end{description}
